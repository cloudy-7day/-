# 修改记录

这个文档用来记录项目每次修改后的说明。以后每次我改代码、网页、脚本或数据结构，都要在这里写清楚：

1. 改了什么
2. 为什么这么改
3. 对你使用有什么影响

## 2026-07-06

### 清理文档中的待确认事项

改了什么：
删除 `CHANGELOG.md` 中所有“需要你确认”段落，并把修改记录格式固定为“改了什么、为什么这么改、对你使用的影响”。

为什么这么改：
你明确要求需要确认的方案只在对话里讨论，不写进方案或修改记录。文档只记录已经确认并实际执行的修改。

对你使用的影响：
以后看修改记录时不会看到开放式待确认事项；需要你做选择时，我会直接在聊天里说明选项、利弊和建议，等你确认后再实现。

### 改成只公开 public 文件夹

改了什么：
新增 `scripts/sync-public.ps1`，把网页需要的文件复制到 `public` 文件夹；启动网页服务时只服务 `public`，不再服务整个项目根目录。每日更新脚本结束后也会同步一次 `public`。

为什么这么改：
项目根目录里有 `.env.local`，里面保存 DeepSeek API key。如果直接把根目录开放给手机，理论上同一网络中的设备可能访问到密钥文件。只开放 `public` 可以让手机看到网页和文章数据，但看不到本机密钥、脚本和其他开发文件。

对你使用的影响：
手机访问方式不变，仍然是浏览器打开电脑 IP 加端口。但底层更安全：手机访问到的是发布目录，不是完整项目目录。

### 增加每日文章归档

改了什么：
网页新增“查看历史日期”下拉框；每日更新脚本现在除了更新 `data/articles.json`，还会把当天内容保存到 `data/archive/YYYY-MM-DD.json`，并维护 `data/archive/index.json` 作为历史日期列表。

为什么这么改：
你希望每天的文章都能按日期保存到新的地方，而不是每天覆盖同一个文件。这样可以回看任意一天的 7 篇文章，也方便以后做复盘。

对你使用的影响：
网页默认仍然显示今天内容；如果有历史归档，页面会出现日期选择，可以切换到某一天的文章。数据层面，每天都会多保存一份日期文件。

### 建立修改说明规则

改了什么：
新增 `CHANGELOG.md`，作为以后所有修改的固定记录位置。

为什么这么改：
你希望每次修改后都能清楚知道“我改了什么、为什么这么改”。如果只写在聊天里，后面很难回看；如果都塞进 README，README 会变得臃肿。所以单独建立修改记录文档。

对你使用的影响：
以后你可以直接看这个文件，了解系统每次变化背后的原因，不需要翻聊天记录。

### 论文筛选规则调整

改了什么：
学术论文筛选改成“权威来源 70 分 + 类型匹配 30 分”。权威来源包括 Nature、Science、Cell、PNAS，以及 Nature Biomedical Engineering、Nature Machine Intelligence、Nature Electronics、Nature Energy、Nature Human Behaviour 等。arXiv 只作为补位来源。

为什么这么改：
你指出学术论文不能用浏览量作为主要标准，因为很多专业方向本来就不会有大众流量。对论文来说，权威来源比热度更重要，所以把权威机构/期刊作为最大权重。

对你使用的影响：
论文栏会优先挑选高可信来源中的应用型突破，而不是网上讨论最多的论文。只有权威索引暂时找不到合适内容时，才回退到 arXiv。
