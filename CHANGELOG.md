# 修改记录

这个文档用来记录项目每次修改后的说明。以后每次我改代码、网页、脚本或数据结构，都要在这里写清楚：

1. 改了什么
2. 为什么这么改
3. 对你使用有什么影响

## 2026-07-07

### 增加文章来源选择规则上下文

改了什么：
新增 `docs/context/article-source-selection-rules.md`，把每天 7 篇内容的来源选择规则整理出来。规则保留你已经定好的国际新闻、AI 应用文章、应用型论文三类结构，并明确 TLDR 权重为 B：只作为候选池之一。

为什么这么改：
你希望后续扩展来源时先从已有规则出发，TLDR 可以参考，但不能让系统变成单一 newsletter 的复刻。独立文档能让后续改脚本时有清晰依据。

对你使用的影响：
当前网页还不会因为这个文档自动变化；它会作为下一步修复和扩展来源链路的标准。后续系统会优先选择公开、稳定、多样、可解释的来源。

### 增加合规爬虫学习上下文

改了什么：
新增 `docs/context/compliant-crawler-learning-context.md`，把后续学习合规爬虫时需要记住的边界、流程、TLDR 参考方式、来源多样性建议和验收标准整理成独立文档。

为什么这么改：
你希望先把“个人学习爬虫但不越过合规红线”的思路导出成上下文，等当前项目完成后再单独练习。这个文档可以帮助后续继续学习时不重新讨论基本边界。

对你使用的影响：
当前网站和每日数据不会因此改变。后续如果要做爬虫练习，可以先打开这个文档作为项目规则；主项目仍然优先使用公开、稳定、可追溯的数据源。

## 2026-07-06

### 将国际新闻从付费墙来源改为开放 RSS 来源

改了什么：
把每日更新脚本里的国际新闻来源从固定的 New York Times RSS，改成开放 RSS 来源池。默认来源包括 BBC World、NPR World 和 The Guardian World；如果以后想加入 TLDR newsletter 或其他可直接阅读的来源，可以在 `.env.local` 里配置 `NEWS_FEED_URLS`，多个 RSS 链接用英文逗号分隔。

为什么这么改：
New York Times 文章经常需要订阅，适合看标题和摘要，不适合做“每天点开就能读”的个人信息库。开放 RSS 来源更稳，也更符合你想要的 TLDR 邮件式阅读体验。

对你使用的影响：
网站上的国际新闻现在会避开付费墙来源，今日数据已重新生成并同步。这个改动不会去绕过订阅墙，只使用公开可访问的 feed 和链接。

### 增加本次工作总结归档文件夹

改了什么：
新增 `work-summary-2026-07-07` 文件夹，里面包含本次工作的总结 `SUMMARY.md`，以及网页、脚本、数据、GitHub Actions 配置和文档的副本。同时把 `work-summary-*` 加入 `.gitignore`。

为什么这么改：

对你使用的影响：
以后你可以直接打开 `D:\111 codex\work-summary-2026-07-07` 回看这次阶段性成果。归档中不包含 `.env.local`，不会暴露 DeepSeek API key。

### 修复推送测试触发被跳过的问题

改了什么：
修改 `.github/workflows/daily-update.yml` 的时间检查逻辑，让 `push` 触发和手动触发一样直接运行更新步骤。

为什么这么改：
第一次推送测试虽然成功启动了工作流，但因为时间检查只允许手动触发和定时触发继续，导致真正的文章更新步骤被跳过，无法验证 `DEEPSEEK_API_KEY` Secret。

对你使用的影响：
这次再次推送后，GitHub Actions 会真正执行文章更新。如果成功，就能证明 GitHub Secret 配置可用。

### 增加自动更新任务的推送测试触发

改了什么：
给 `.github/workflows/daily-update.yml` 增加了 `push` 触发条件。当工作流文件或 `scripts/update-daily.ps1` 更新并推送到 `main` 时，GitHub Actions 会自动运行一次。

为什么这么改：
GitHub Secret 无法直接读取，只能通过运行工作流来验证。增加这个触发条件后，本次推送就能立刻测试 `DEEPSEEK_API_KEY` 是否能被云端任务正确使用。

对你使用的影响：
以后普通文章数据更新不会触发循环运行；只有工作流文件或更新脚本发生变化时才会额外测试一次。每天洛杉矶时间 8 点的定时运行不受影响。

### 增加 GitHub 自动更新任务

改了什么：
新增 `.github/workflows/daily-update.yml`。它会在 GitHub 上定时运行 `scripts/update-daily.ps1`，生成当天文章数据和日期归档，并把更新后的 `data/articles.json` 与 `data/archive` 提交回仓库。

为什么这么改：
你希望网站更稳定，不依赖电脑 IP 和电脑开机状态。GitHub Pages 可以提供固定访问地址，GitHub Actions 可以在云端每天自动更新内容。

对你使用的影响：
后续只要在 GitHub 仓库里设置 `DEEPSEEK_API_KEY` Secret，并开启 GitHub Pages，手机就可以访问固定 GitHub Pages 地址。电脑关机不会影响网页访问。

### 清理文档中的待确认事项

改了什么：
删除 `CHANGELOG.md` 中所有“需要你确认”段落，并把修改记录格式固定为“改了什么、为什么这么改、对你使用的影响”。

为什么这么改：
你明确要求需要确认的方案只在对话里讨论，不写进方案或修改记录。文档只记录已经确认并实际执行的修改。

对你使用的影响：
以后看修改记录时不会看到开放式待确认事项；需要你做选择时，我会直接在聊天里说明选项、利弊和建议，等你确认后再实现。

### 改成只公开 public 文件夹

改了什么：
新增 `scripts/sync-public.ps1`，把网页需要的文件复制到 `public` 文件夹；启动网页服务时只服务 `public`，不再服务整个项目根目录。每日更新脚本结束后也会同步一次 `public`。

为什么这么改：
项目根目录里有 `.env.local`，里面保存 DeepSeek API key。如果直接把根目录开放给手机，理论上同一网络中的设备可能访问到密钥文件。只开放 `public` 可以让手机看到网页和文章数据，但看不到本机密钥、脚本和其他开发文件。

对你使用的影响：
手机访问方式不变，仍然是浏览器打开电脑 IP 加端口。但底层更安全：手机访问到的是发布目录，不是完整项目目录。

### 增加每日文章归档

改了什么：
网页新增“查看历史日期”下拉框；每日更新脚本现在除了更新 `data/articles.json`，还会把当天内容保存到 `data/archive/YYYY-MM-DD.json`，并维护 `data/archive/index.json` 作为历史日期列表。

为什么这么改：
你希望每天的文章都能按日期保存到新的地方，而不是每天覆盖同一个文件。这样可以回看任意一天的 7 篇文章，也方便以后做复盘。

对你使用的影响：
网页默认仍然显示今天内容；如果有历史归档，页面会出现日期选择，可以切换到某一天的文章。数据层面，每天都会多保存一份日期文件。

### 建立修改说明规则

### 论文筛选规则调整

改了什么：
学术论文筛选改成“权威来源 70 分 + 类型匹配 30 分”。权威来源包括 Nature、Science、Cell、PNAS，以及 Nature Biomedical Engineering、Nature Machine Intelligence、Nature Electronics、Nature Energy、Nature Human Behaviour 等。arXiv 只作为补位来源。

为什么这么改：
你指出学术论文不能用浏览量作为主要标准，因为很多专业方向本来就不会有大众流量。对论文来说，权威来源比热度更重要，所以把权威机构/期刊作为最大权重。

对你使用的影响：
论文栏会优先挑选高可信来源中的应用型突破，而不是网上讨论最多的论文。只有权威索引暂时找不到合适内容时，才回退到 arXiv。
