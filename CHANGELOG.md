# 修改记录

## 2026-07-16

### 增加国内要闻并将每日内容扩展为 9 篇
改了什么：

每日内容从 7 篇扩展为 9 篇：3 篇国内要闻、2 篇国际政治或金融新闻，以及原有 2 篇 AI 应用和 2 篇论文/AI 补位内容。国内候选接入中国新闻网、新华网、人民网和 China Daily 官方 RSS，外交部安全提醒作为国际重大事件补充；新增可测试的重要性排序、娱乐软文排除和 24/48 小时时效规则。网页把原「天下异闻」改为「天下要闻」，在同一入口分组显示国内和国际新闻。
为什么这样改：

原有新闻全部偏国际，日常阅读与国内现实联系较弱。固定 3 篇国内、2 篇国际可以同时覆盖国内政策、灾害、科学和社会变化，以及国际政治与金融信号；规则筛选也避免为了凑数量混入娱乐内容。
对使用的影响：

每天发布前必须验证 9 篇结构。单个 RSS 失败时会重试并从其他公开来源补足；严格重大新闻不足时允许宏观经济、产业政策和普通社会新闻补位。最终仍无法凑齐 3 篇国内和 2 篇国际时，本次发布失败并保留上一份完整数据。旧的 7 篇历史归档不会被改写，仍可正常查看。

## 2026-07-14

### 修复论文自动总结与 09:00 内容恢复检查

改了什么：
每日条目新增 `summarySource`，每日数据新增 `updateStatus` 和基于 7 篇 URL 的 `contentFingerprint`。DeepSeek 缺失或临时失败时，不再发布通用占位文案，而是发布明确标注的公开来源摘录；DeepSeek 恢复后只升级总结，不重新选择当天文章。GitHub Actions 现在覆盖洛杉矶时间约 08:07 和 09:07，09:00 恢复检查会验证当天日期、7 篇结构、发布文件与归档一致性、URL 指纹及总结状态。发布过程改为校验后写临时文件，并在失败时回滚。

为什么这样改：
2026-07-14 的一次 push 更新在未取得可用 `DEEPSEEK_API_KEY` 时仍显示成功，并把全部内容覆盖成 `Local fallback`；旧校验只检查摘要非空，旧工作流也把“当天归档存在”误当作更新完成。这会造成任务是绿色的，但论文没有真实总结，或者页面仍是旧内容。

对使用的影响：
约 08:07 首次生成；约 09:07 会检测内容是否真的变化。当天缺失、结构错误、发布未同步或 7 篇 URL 与前一天完全相同时会整批重新生成；只有总结降级时只重试 DeepSeek。临时摘录会在中英文页面明确提示，正式总结不会显示提示。

## 2026-07-13

### 补齐云端抓取依赖并防止不完整数据发布

改了什么：
GitHub Actions 现在显式配置 Python 3.12 并安装固定版本 `pypdf==6.10.2`，把受限的 `GITHUB_TOKEN` 传给 GitHub 搜索请求，限制同一时间只运行一个每日更新任务，并设置 30 分钟总超时。RSS、GitHub、Crossref、arXiv、PDF 下载和 DeepSeek 请求各自失败时会重试一次；整个更新仍失败时，工作流等待 30 秒再重试一次。主脚本统一使用洛杉矶日期，并在覆盖 `data/articles.json` 前验证日期、分类组合、唯一 ID/URL、发布时间、中英文摘要、论文可读状态和双语论文卡片。排队的定时任务会在执行时检出最新 `main`，避免基于旧提交重复生成。

为什么这么改：
此前精确到小时的时间门禁已经改为按日期幂等，但云端仍没有显式安装 PDF 提取依赖，GitHub 搜索也使用未认证请求，外部服务短暂不可用时没有请求级重试。两个延迟 cron 还可能读取同一个旧提交。上述问题会造成论文抓取失败、API 限流、重复生成或当天数据无法及时提交。发布前校验可以避免失败结果覆盖上一份可用数据。

对你使用的影响：
GitHub 定时任务即使延迟启动也会检查当天归档并执行；短暂网络故障会自动重试一次。只有结构完整的 7 篇内容会发布到 GitHub Pages，失败时保留上一份可用数据，并在 Actions 中显示明确失败而不是假成功。

## 2026-07-07

### 接入论文公开全文与 PDF 总结规则

改了什么：
论文每日推荐改为只保留有公开全文入口的 AI、脑机、芯片、能源方向论文；arXiv 论文会指向 PDF 链接，并在更新脚本中尝试提取 PDF 文本后交给 DeepSeek 生成通俗论文卡片。若 PDF 文本提取失败，论文不会进入每日推荐，并由 AI 应用文章补位。

对你使用的影响：

### 调整 AI 应用文章筛选为证据链规则

改了什么：
新增 `scripts/article-selection.ps1` 和 `scripts/test-ai-selection.ps1`，把 AI 候选拆成“概念解释类”和“应用创新类”。AI 应用现在必须满足 3 个月硬性时限、至少 2 个证据锚点、至少 1 个一手锚点；候选来源新增 GitHub Search 和英文工程圈公开 feed，并保留 HN 讨论作为辅助信号。网页会在 AI 内容里展示证据链，并且只对应用创新类显示“为什么它可能不能成功”，其他内容显示“关键看点”。

为什么这么改：

对你使用的影响：
每天 AI 栏会更偏向近期、可追溯、和实际应用强相关的内容。若是前沿应用项目，会保留反方分析；若是概念解释或行业变化，会以简介和关键看点为主，不再强行套“为什么失败”的框架。

### 改成只公开 public 文件夹

改了什么：
新增 `scripts/sync-public.ps1`，把网页需要的文件复制到 `public` 文件夹；启动网页服务时只服务 `public`，不再服务整个项目根目录。每日更新脚本结束后也会同步一次 `public`。

为什么这么改：
项目根目录里有 `.env.local`，里面保存 DeepSeek API key。如果直接把根目录开放给手机，理论上同一网络中的设备可能访问到密钥文件。只开放 `public` 可以让手机看到网页和文章数据，但看不到本机密钥、脚本和其他开发文件。

对你使用的影响：
手机访问方式不变，仍然是浏览器打开电脑 IP 加端口。但底层更安全：手机访问到的是发布目录，不是完整项目目录。

### 论文筛选规则调整

改了什么：
学术论文筛选改成“权威来源 70 分 + 类型匹配 30 分”。权威来源包括 Nature、Science、Cell、PNAS，以及 Nature Biomedical Engineering、Nature Machine Intelligence、Nature Electronics、Nature Energy、Nature Human Behaviour 等。arXiv 只作为补位来源。

为什么这么改：
你指出学术论文不能用浏览量作为主要标准，因为很多专业方向本来就不会有大众流量。对论文来说，权威来源比热度更重要，所以把权威机构/期刊作为最大权重。

对你使用的影响：
论文栏会优先挑选高可信来源中的应用型突破，而不是网上讨论最多的论文。只有权威索引暂时找不到合适内容时，才回退到 arXiv。
