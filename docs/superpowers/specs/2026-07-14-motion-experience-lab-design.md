# 山海经网站动效体验实验室设计

日期：2026-07-14  
状态：待用户通过动态演示审查

## 1. 目标

为「山海经的漏网之鱼」制作一个使用真实页面内容的 A/B/C 动效体验实验室，让用户先实际滚动、点击和切换，再选择最终动效。设计不能只以文字或静态截图交付；本次以及后续每次视觉设计结束时，都必须同时提供可操作网页或动态演示作为审查依据。

本次同时解决现有界面的具体使用问题：首屏一次滚轮后仍停在半途、全站字体缺少统一气质、顶部与栏目文案冗余、分类卡片提示重复、文章详情被人为拆成两页。

## 2. 不变范围

- 保留原生 HTML、CSS、JavaScript 与 hash 路由，不改变每日 JSON 数据结构。
- 不修改 DeepSeek、自动更新、GitHub Actions、归档生成或 GitHub Pages 地址。
- 保留中文/English、历史归档、明暗主题、安全外链、论文摘要/PDF 双入口。
- 动效必须是渐进增强；不支持新 API、启用减少动态偏好或脚本加载失败时，内容仍完整可用。
- 不把外部网站的受保护视觉素材复制到本站，只借鉴公开的交互机制。

## 3. 外部研究结论

### 3.1 原生滚动吸附与滚动时间线

CSS Scroll Snap 可在一次滚动操作结束后把视口吸附到完整场景，避免停在两屏之间。CSS Scroll-driven Animations 可把动画直接绑定滚动时间线，减少主线程 `scroll` 监听造成的卡顿风险。本站的 A 方案使用这两个机制，并保留现有 JavaScript 进度作为兼容降级。

参考：

- https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Scroll_snap
- https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Scroll-driven_animations

### 3.2 Lenis 惯性滚动

Lenis 通过统一的动画帧循环平滑滚轮、触控板和触摸输入，适合需要轻微视差的沉浸式页面。官方同时说明原生 CSS scroll snap 需要配合单独的 Lenis Snap 插件，因此任何惯性方案都不能再叠加浏览器强制吸附。V2 审查页最终用一个更小的可取消场景控制器复现了用户选中的手感；生产实现沿用已审查控制器，不再额外加入 Lenis 依赖。

参考：https://github.com/darkroomengineering/lenis

### 3.3 View Transition

View Transition API 可在 DOM 状态切换前后拍摄快照，并让同名元素从旧位置连续移动到新位置。本站本来就是 hash 单页应用，适合让栏目标题或文章标题从卡片自然延续到下一页；不支持该 API 的浏览器直接切换，不影响导航。

参考：

- https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API
- https://web.dev/learn/css/view-transitions-spas

### 3.4 中文字体

正文候选采用本地托管的「霞鹜文楷屏幕阅读版」。该字体专为 PC 与 Android 屏幕阅读调整字重和度量，并使用 SIL Open Font License 1.1。标题继续使用宋体/明朝体栈，以免整站变成手写风格。字体文件必须随 GitHub Pages 发布，不依赖远程字体 CDN。

参考：https://github.com/lxgw/LxgwWenKai-Screen

## 4. 共同界面简化

三种动效模式只比较运动方式，信息架构和字体保持一致：

- 删除顶部居中的「闻一之间的空间」，顶部改为左侧站名、右侧时间/语言/主题。
- 首页三张栏目卡删除「观世界变局」「察器物新生」「究万物之理」等小字。
- 分类页标题上方不再显示同类红色小字。
- 分类卡片删除「启封细读」；整张卡片仍可点击，并保留清晰的悬停与键盘焦点反馈。
- 详情页删除「卷 XX · 条目」、`1 / 2`、翻页按钮、`BRIEF` 和 `ASSOCIATIONS` 小标题。
- 「这件事在说什么」与「顺手想一想」合并为同一页面。
- 桌面详情采用不对称双栏：摘要约占 58%，延伸联想约占 42%；插画作为低对比留白点景，不再复制到两页。
- 手机详情自然变为单列：摘要、延伸联想、来源入口依次排列。

## 5. A/B/C 动效模式

体验实验室提供固定但低存在感的 `A / B / C` 切换器，并把选择同步到 `?motion=a|b|c` 与 `localStorage`。切换模式后回到首屏开头，避免不同滚动位置影响比较。

### A. 原生快切（默认与推荐）

- 首屏滚动行程从约 `120vh` 压缩到 `42vh`，完整容器高度约 `142vh`。
- 首次向下滚动后，浏览器以 `scroll-snap-type: y proximity` 吸附到栏目场景。
- 标题退场和栏目进场使用滚动时间线；兼容模式继续写入 `--intro-progress`。
- 背景只做 `1 → 1.035` 的轻推近，标题上移不超过 `12vh`，避免大幅晃动。
- 栏目/文章导航使用 240–320ms 的共享标题 View Transition。
- 目标手感：反应最快、最接近原生网站，不出现滚轮拖泥带水。

### B. 惯性丝滑

- 使用审查页已经验证的单一可取消场景控制器，不新增滚动依赖。
- 一次明确的向下滚轮、触摸上滑或“进入”操作触发 `1150ms` 场景过渡；新输入先取消旧动画。
- 标题在前 46% 完整上飞淡出；栏目卡从 `34vh` 下方错峰飞入，并由低饱和、轻模糊恢复正常颜色。
- 场景控制器在同一动画帧内更新滚动位置和视觉帧，不同时启用 CSS scroll snap。
- 页面切换使用 360ms 淡入加 16px 纵向位移。
- 目标手感：保留用户选定的沉浸感，同时确保停在可点击的完整栏目场景。

### C. 直接极简

- 首页不保留滚动进度舞台；首个明确向下滚轮、触摸上滑或「进入」按钮触发场景切换。
- 主标题在 320–380ms 内缩小、淡出，栏目入口同时淡入并上移 20px。
- 切换完成后页面立即恢复普通原生滚动；向上回到顶部时可返回标题场景。
- 栏目与详情采用 220ms 纯交叉淡化，不做共享元素位移。
- 目标手感：最快、最简洁，牺牲一部分画卷展开的仪式感。

## 6. 动态审查页

### 6.1 本次设计演示

设计审查页必须包含：

- 使用当前站点的《四季山水》背景、真实站名、三个真实栏目标题和一篇真实文章示例。
- A/B/C 即时切换，显示当前方案名称和一句手感提示。
- 可实际滚轮完成首屏切换，可点击栏目进入分类，可点击文章进入合并后的单页详情。
- 提供「重置体验」按钮，把滚动和路由恢复到初始状态。
- 不连接生产数据写入，不修改线上页面；它只用于设计审查。

### 6.2 后续固定规则

以后每份视觉或交互设计文档的验收章节必须声明一个对应审查产物：

- 交互设计：可点击 HTML 原型或真实前端预览。
- 动效设计：可滚动/点击的动态演示，不能只交 GIF 截图。
- 纯视觉设计：至少提供浏览器可打开的高保真页面；若涉及响应式，包含桌面与手机切换。
- 审查产物必须给出明确访问方式，并保留到用户确认最终方案。

## 7. 状态与数据流

- `motionMode`: `a | b | c`，优先读取 URL，其次读取 `localStorage`，默认 `a`。
- `homeScene`: `hero | themes`，由当前模式的滚动或触发器决定。
- 路由仍由 `location.hash` 决定；动效包装只负责渲染前后过渡，不改变路由解析。
- 详情合并后删除 `detailPage` 状态和 `setDetailPage()`。
- 模式切换必须先销毁上一模式的监听器/动画实例，再初始化下一模式，避免重复滚动处理。

## 8. 无障碍与失败处理

- `prefers-reduced-motion: reduce` 下三种模式统一直接显示栏目，并关闭平滑滚动和 View Transition 动画。
- A/B/C 切换器使用真实按钮，提供 `aria-pressed` 和键盘焦点样式。
- C 模式不能只监听滚轮，同时提供触摸与按钮入口。
- 场景脚本加载或执行失败时直接回退普通文档流，不隐藏栏目入口。
- 字体加载失败时回退现有宋体/微软雅黑栈，不隐藏正文。
- 动画中不锁死页面超过 450ms，不阻止浏览器返回、键盘导航或正文选择。

## 9. 测试与验收

### 自动化契约

- A 模式首屏行程不超过 `45vh`，并包含 proximity snap 与兼容进度路径。
- B 模式同时只能存在一个 `requestAnimationFrame` 场景动画，新动画开始前必须取消旧动画。
- C 模式具备 wheel、touch 和按钮三种入口。
- 三模式均遵循 reduced-motion。
- 页面不再输出已删除的口号、小红字、「启封细读」、详情进度和翻页按钮。
- 详情 DOM 同时包含摘要与延伸联想，且不再使用 `hidden` 分页。
- 字体文件存在于发布资产目录，CSS 使用 `font-display: swap`。
- `public/` 与根目录前端文件一致。

### 人工体验验收

- 桌面鼠标一次正常向下滚动能稳定到达完整栏目场景。
- 触控板慢滚时动画仍跟手，快速滑动不会停在无法点击的中间态。
- 手机无需滚轮即可进入栏目。
- 用户能在动态审查页依次体验 A、B、C，并明确选择最终方案或提出修改。
- 最终生产实现必须在本地动态预览和 GitHub Pages 真机访问中各验证一次。

## 10. 交付顺序

1. 先交付不修改生产站的动态设计审查页，供用户体验 A/B/C。
2. 根据用户反馈调整演示，直到选定一个最终方案。
3. 再把共同简化、字体和选定动效写入生产站。
4. 运行自动化、桌面/手机视觉检查并同步 `public/`。
5. 推送 GitHub Pages，验证真实线上页面后完成。

## 11. V2 动效修订（依据首次体验反馈）

首次动态审查确认三个方案的视觉冲击不足，A 存在抖动，B/C 节奏过快。第二版统一改为一条可取消的场景时间线：

- 标题在时间线前 46% 内由完整显示过渡为透明，并从原位上飞至 `-38vh`；同时轻微缩小至 `0.93`、增加不超过 `4px` 的模糊，确保是完整离场而非停在半空。
- 三张栏目卡从 `34vh` 下方依次飞入，起始饱和度为 `0.08`、模糊为 `5px`，到达画面中心时恢复正常颜色、清晰度和比例；卡片之间保留约 7.5% 的错峰，增强层次与冲击力。
- A 不再把浏览器 CSS scroll snap 与自定义滚动动画叠加；所有滚轮输入只驱动一个可取消的 `requestAnimationFrame` 控制器，新输入会先取消旧动画，消除多个写入者争夺滚动位置造成的抖动。
- 三个方案的单次场景时长调整为 A `760ms`、B `1150ms`、C `920ms`。A 保持利落，B/C 明显放慢，三者仍保留不同手感供审查。
- 动态审查页升级为 V2，并增加静态契约测试，检查标题完整离场、卡片颜色恢复、单控制器取消机制、无 CSS scroll snap 冲突以及三种节奏差异。

第二版仍只用于设计审查；用户选定并确认最终手感前，不写入生产站点。

## 12. 最终选择与生产文案规则

用户在 V2 动态审查页中选定 B 方案。生产实现采用 B 的 `1150ms` 沉浸节奏、标题完整上飞淡出、栏目卡错峰飞入并由低饱和恢复正常颜色的视觉语言；滚动仍由单一可取消的场景控制器负责，不重新引入 CSS scroll snap 与脚本争夺滚动位置的问题。

首页及分类页继续执行前述简化，并新增以下确定规则：

- 删除首页副标题“异闻、机巧与格物，藏入每日七篇。”，不以替代口号填补留白。
- 文章数据新增独立的 `highlight` 字段；它只服务分类列表的标题下方短句，不替代详情页使用的 `summary`。
- 中文界面的 `highlight` 优先采用来源正文中最有概括性或最吸引人的完整句子的忠实中文表达；英文界面的 `translations.en.highlight` 优先保留来源英文原句。来源不是英文时，英文界面使用忠实英文翻译。
- 来源句子缺少上下文、只是导航或 RSS 占位信息、过长或无法独立成立时，允许生成一句克制的编辑概括。
- 中文短句目标为 25–55 个汉字，英文目标为 12–30 个单词；前端最多显示两行，但数据层保存完整短句，不用 CSS 截断替代内容质量控制。
- 禁止“本文介绍”“文章指出”“值得阅读”“这篇论文提出”等模板开头，也禁止虚构来源中没有的事实。
- DeepSeek 可用时由同一次结构化分析同时生成完整摘要和双语 `highlight`；DeepSeek 不可用时，从可追溯的 `sourceExcerpt` 选择第一条可独立成立的完整句子。
- 旧归档缺少 `highlight` 时，前端从对应语言的 `summary` 选择第一条非模板化完整句作为兼容回退，不批量重写历史归档。
- 08:00 主生成、09:00 内容检测后的补偿生成以及降级摘要升级都必须写入并验证 `highlight`，避免次日或补偿运行恢复旧文案。

验收时，当前七篇文章的分类列表必须全部显示独立短句；不得显示空值、完整多段摘要、来源故障说明或模板化“值得阅读”文案。动态预览与 GitHub Pages 真机页面都要验证中文、英文以及旧归档回退。
