# 栏目新闻列表比例调整设计

## 目标

让栏目新闻列表在浏览器 100% 缩放时接近当前 75% 缩放的视觉密度，同时不改变首页、收藏页和文章详情页。

## 已确认范围

- 只调整 `#/category/*` 栏目页的页头、新闻标题、摘要、间距和列表宽度比例。
- 使用栏目页专属 CSS 尺寸，不使用 `transform: scale()` 或非标准 `zoom`，避免模糊、空白和点击区域错位。
- 删除栏目列表中每条新闻右侧的收藏按钮。
- 顶部站点标题栏的收藏页入口保持不变。
- 文章详情页收藏按钮保持不变，用户仍可进入文章后新增或取消收藏。
- 将“往日卷册 / 当前日期”切换从栏目页大标题右侧移到新闻列表右侧动作列，只显示一次，并与第一条新闻的来源和发布日期行水平对齐。
- 首页、收藏列表、收藏详情和普通文章详情的字号与布局不变。

## 结构方案

栏目页渲染时，在新闻索引顶部加入独立的归档控件层。该层与新闻条目使用相同的主内容列和右侧动作列网格，因此归档控件能够稳定落在原收藏按钮所在列，并与第一条元数据行对齐。新闻条目在栏目页不再渲染 `index-favorite`，收藏页仍继续渲染该按钮。

栏目页增加专属样式作用域，例如 `.category-view`，将标题、索引正文和纵向节奏调整到现有值约 75% 的视觉尺寸。移动端不机械缩小到 75%，而是保留可读性下限并让归档控件自然换行。

## 验证

- 1440px 与 2048px 桌面视口：栏目页显示至少三条新闻时比例接近原页面浏览器 75% 缩放效果。
- 归档控件位于第一条新闻右侧并与日期元数据对齐。
- 栏目列表不存在收藏按钮；详情页和顶部收藏入口仍存在。
- 首页、收藏页和详情页的关键字号契约不变。
- 620px 移动视口无横向溢出，日期切换仍可操作。
