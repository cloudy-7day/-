# 栏目新闻列表层级再平衡设计

## 选择

采用“满行布局 + 中等标题 + 放大小字”的平衡方案。

不采用完全恢复旧版，因为旧版标题过大；不保留当前 75% 紧缩版，因为列表过窄、大小字反差过强。新方案取两者中间值，并优先恢复横向张力。

## 版式

- 栏目新闻索引恢复约 `1400px` 的满行宽度，让长标题尽量横向展开。
- 栏目主标题采用介于旧版和紧缩版之间的尺寸。
- 新闻标题采用中等尺寸：明显大于正文，但不再压满屏幕高度。
- 来源和发布日期从约 `11px` 提升到 `13px`。
- 摘要从紧缩版 `11px` 提升到约 `15px–17px`，行距同步增加。
- 新闻序号略微放大，保持与元数据同一视觉层级。
- 卡片高度和上下留白恢复到中间值，使三条新闻仍有呼吸感。

## 控件

- “往日卷册 / 当前日期”继续位于第一条新闻右侧，与来源日期行对齐。
- 第一条新闻为日期控件预留宽度，后续新闻使用完整行宽。
- 栏目列表不显示逐条收藏按钮。
- 顶部收藏入口和文章详情收藏按钮保持不变。

## 作用范围与验证

- 只修改 `#/category/*` 栏目页；首页、收藏页和详情页不变。
- 桌面端检查三条新闻是否重新形成横向铺陈，且标题、元数据、摘要呈连续三级层次。
- 390px 移动端保持无横向溢出，日期控件在列表上方自然换行。
